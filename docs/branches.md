# BRANCHES
branches são como versões do projeto. Eles são úteis para que cada um consiga trabalhar de forma independente no projeto.

## Nome das branches
Os nomes das branches devem indicar o motivo da sua criação. Se ele foi criado para a pessoa X alterar o projeto, ele pode ser chamado de "X". Se ele foi criado para implementar a funcionalidade Y, ele pode ser chamado de "featureY".
## Estrutura das branches
Não se deve criar estruturas complexas de branches com infinitas ramificações. Quando você estiver trabalhando com versões diferentes, tente utilizar o menor número de branches possíveis, evitando assim uma complicação desnecessária para o projeto. Separe elas somente pelo que é essencial.
